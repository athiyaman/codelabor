Nomalization vs. Denomalization

Layering vs. Delayering

각 단계별 전략 변화의 사유는 합당하나 아키텍처 전체를 평가할 때
+, -, 0 가 되는 아키텍처 상의 상쇄가 존재한다면
불필요한 레이어, 패턴을 제거할 수 있다.

예:
DAO Manager는 DB Table에 대해 1:1로 DAO, VO를 생성
Table 변경 시마다 새로 생성
new 로 생성하므로 POJO로 관리

Service도 DAO로 1:1 매핑되는 단순 구조이고 
Table 변경에 직접적인 영향을 받을 경우
DAO의 존재 이유는 상실됨

new로 생성하여 Spring Bean으로 관리되지 않는 경우, 
AOP 적용도 불가하여 SQL Injection, Authorization 적용 역시 쉽지 않음

더구나 DAO Manager가 사용하는 Any Server가 축적한 Meta 데이터는
좀 처럼 활용도가 없음에도 불구하고 
시스템 장애 시, 개발을 하지 못하게 되는 부작용이 있음
